# Configuration reference

`config.toml` controls default COMMUNE library paths for zone-aware workflows.

## Config lookup

Configuration is loaded from the first matching source:

1. `DEDUPE_CONFIG` environment variable (explicit override)
2. `./config.toml` (repo/project root)
3. `~/.config/dedupe/config.toml`

```toml
[library]
name = "COMMUNE"
root = "/Volumes/COMMUNE"

[library.zones]
inbox = "00_INBOX"
staging = "10_STAGING"
accepted = "20_ACCEPTED"
rejected = "90_REJECTED"
yate_db = "_yate_db"
```

- **root** – absolute path to the COMMUNE library.
- **inbox** – raw arrivals; excluded from automated processing.
- **staging** – active Yate curation zone; metadata writes allowed here.
- **accepted** – canonical library visible to Roon; read-only for dedupe.
- **rejected** – explicitly excluded; never scanned or hashed.
- **yate_db** – Yate database folder; always ignored.

Other configuration files (CSV manifests, playlists, etc.) are generated by
individual workflows and are not required inputs.

Environment variables are still used for API credentials in other parts of the
repository; this refactor does not remove or bypass any existing security
controls.
