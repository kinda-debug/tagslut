# Configuration reference

`config.toml` controls default paths for the synchronisation workflow.

## Config lookup

Configuration is loaded from the first matching source:

1. `DEDUPE_CONFIG` environment variable (explicit override)
2. `./config.toml` (repo/project root)
3. `~/.config/dedupe/config.toml`

```toml
[paths]
root = "/Volumes/dotad/MUSIC"
quarantine = "/Volumes/dotad/Quarantine"
garbage = "/Volumes/dotad/Garbage"
```

- **root** – absolute path to the primary music library. When omitted the
  synchronisation CLI falls back to `/Volumes/dotad/MUSIC`.
- **quarantine** – root of the quarantine review area. The commands walk the
  directory recursively, so nested artist/album folders are automatically
  included.
- **garbage** – archive of confirmed duplicates retained for healthier copies.
  These are also scanned recursively when you point commands at them.

Other configuration files (CSV manifests, playlists, etc.) are generated by
individual workflows and are not required inputs.

Environment variables are still used for API credentials in other parts of the
repository; this refactor does not remove or bypass any existing security
controls.
