## Supported Language Versions

## Installation

If you use Maven, place the following within the _dependency_ tag in your `pom.xml` file:

```XML
<dependency>
    <groupId>com</groupId>
    <artifactId>searchservicesdk</artifactId>
    <version>1.0.0</version>
</dependency>
```

If you use Gradle, paste the next line inside the _dependencies_ block of your `build.gradle` file:

```Gradle
implementation group: com, name: searchservicesdk, version: 1.0.0
```

If you use JAR files, package the SDK by running the following command:

```shell
mvn compile assembly:single
```

Then, add the JAR file to your project's classpath.

## Setting a Custom Timeout

You can set a custom timeout for the SDK's HTTP requests as follows:

```java
import com.searchservicesdk.SearchServiceSdk;
import com.searchservicesdk.config.SearchServiceSdkConfig;

public class Main {

  public static void main(String[] args) {
    SearchServiceSdkConfig config = SearchServiceSdkConfig.builder().timeout(10000).build();
    SearchServiceSdk searchServiceSdk = new SearchServiceSdk(config);
  }
}

```

### OneOf models

OneOf models represent any one of several distinct models, allowing a single strongly typed model to encompass multiple interchangeable schemas used by the API.

While using this SDK, you may need to instantiate a OneOf model to create a request or else handle one from a response.

#### Instantiating OneOf models

While regular models usually have a builder class that can be used to instantiate them, OneOf models don't.
Instead, they have a static factory method for each of it's possible types, prefixed with `of` followed by the type name.

To instantiate a OneOf model, you can use the static factory method for the type you want to instantiate.
For example, to instantiate a `ParamType` that can represent a `TypeA` or `TypeB`, you can use the following code:

```java
ParamType paramTypeWithTypeA = ParamType.ofTypeA(typeA);

ParamType paramTypeWithTypeB = ParamType.ofTypeB(typeB);

```

#### Handling OneOf models from a response

When handling a response from the API, you may need to deal with a OneOf model.

You can use the `switchCase` method on the OneOf model to provide a `Consumer` for each of the possible types:

```java
ParamType paramType = ParamType.ofTypeA(typeA);

paramType.switchCase(
  typeA -> System.out.println("TypeA"),
  typeB -> System.out.println("TypeB")
);
```

Or you can use the `match` method to provide a `Function` for each of the possible types, and get a value out:

```java
ParamType paramType = ParamType.ofTypeA(typeA);

String message = paramType.match(
  typeA -> "TypeA",
  typeB -> "TypeB"
);

System.out.println(message);
```

You can also use the provided getters to get a specific type value out.
Just note that the getters will return `null` if the contained type is not the expected one:

```java
ParamType paramType = ParamType.ofTypeA(typeA);

TypeA typeA = paramType.getTypeA();

TypeB typeB = paramType.getTypeB(); // null

```

If you just want the contained value, regardless of the type, you can use the `getValue` method, which returns an `Object` type.
