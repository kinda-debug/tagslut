# Technical Whitepaper: Personal Qobuz Metadata Tagger

## Architectural Design Document

**Version**: 1.0 | **Date**: 2026-03-29 | **Classification**: Technical Reference

## Executive Summary

This whitepaper provides a production-ready roadmap for building a Personal Qobuz Metadata Tagger—a tool that extracts high-quality metadata from Qobuz’s streaming service using an active personal subscription. The architecture leverages dynamic credential extraction from Qobuz’s web player JavaScript bundles, implements session spoofing for authenticated access, and embeds metadata into local audio files (FLAC/MP3) according to industry standards.

The project addresses a critical gap: Qobuz does not offer public self-service developer accounts, necessitating reverse-engineering approaches validated by successful open-source projects including `streamrip`, `qobuz-dl`, `qobuz-api-rust`, and `QobuzApiSharp`.

## 1. Credential Acquisition & Session Management

### 1.1 Dynamic Credential Extraction Architecture

Qobuz’s API requires two application-level credentials for any request: `APP_ID` and `APP_SECRET`. As of late 2024 (web player v7.2+), these credentials are dynamically bound to user authentication tokens—using hardcoded credentials from older projects with newly generated tokens results in `InvalidAppSecretError` responses .

The recommended approach extracts credentials directly from Qobuz’s public-facing JavaScript bundles at runtime.

#### 1.1.1 Extraction Implementation (Python)

```python
# get_keys.py - Dynamic credential extractor
# Based on vitiko98/qobuz-dl implementation
# Source: https://gist.github.com/vitiko98/bb89fd203d08e285d06abf40d96db592

import re
import requests
from base64 import b64decode
from typing import Dict, Optional

class QobuzCredentialExtractor:
    """Extracts APP_ID and APP_SECRET from Qobuz web player bundles."""
    
    QOBUZ_WEB_URL = "https://play.qobuz.com"
    _BUNDLE_URL_REGEX = r'src="([^"]+bundle\.js)"'
    _APP_ID_REGEX = r'appId\s*:\s*"(\d+)"'
    _SEED_TIMEZONE_REGEX = r'initialSeed\("([^"]+)",window\.utimezone\.([^)]+)\)'
    
    def extract(self) -> Dict[str, str]:
        # 1. Fetch main page to locate bundle.js
        main_page = requests.get(self.QOBUZ_WEB_URL).text
        bundle_url_match = re.search(self._BUNDLE_URL_REGEX, main_page)
        bundle_url = f"https://play.qobuz.com{bundle_url_match.group(1)}"
        
        # 2. Download and parse bundle.js
        bundle_js = requests.get(bundle_url).text
        
        # 3. Extract APP_ID
        app_id = re.search(self._APP_ID_REGEX, bundle_js).group(1)
        
        # 4. Extract secrets using seed/timezone patterns
        seed_matches = re.findall(self._SEED_TIMEZONE_REGEX, bundle_js)
        secrets = {}
        for seed, tz in seed_matches:
            # Decode base64-encoded secret
            decoded = b64decode(seed.encode()).decode('utf-8')
            secrets[tz] = decoded
            
        return {"app_id": app_id, "secrets": secrets}
```

### 1.2 User Authentication Flow

After obtaining `APP_ID` and `APP_SECRET`, the tagger must authenticate as a specific user using credentials from an active Qobuz subscription.

**Critical Constraint**: The `user_auth_token` generated by an `APP_ID` is strictly bound to that ID and its corresponding secret. You cannot mix credentials from different extraction sessions .

#### 1.2.1 Authentication Implementation

```python
class QobuzAuthenticator:
    def __init__(self, app_id: str, app_secret: str):
        self.app_id = app_id
        self.app_secret = app_secret
        self.session = requests.Session()
        self.session.headers.update({
            "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:133.0) Gecko/20100101 Firefox/133.0",
            "X-App-Id": app_id,
            "App-Secret": app_secret  # Note: Some implementations use "App-Secret"
        })
    
    def login_with_email(self, email: str, password_md5: str) -> str:
        """Authenticate using email and MD5-hashed password."""
        payload = {
            "app_id": self.app_id,
            "email": email,
            "password": password_md5
        }
        response = self.session.post(
            "https://www.qobuz.com/api.json/0.2/user/login",
            data=payload
        )
        return response.json()["user"]["auth_token"]
    
    def login_with_token(self, user_id: str, auth_token: str) -> None:
        """Authenticate using pre-existing token."""
        self.user_auth_token = auth_token
        self.session.headers["X-User-Auth-Token"] = auth_token
        # Token validity is primary - user_id is secondary 
```

### 1.3 Legal & Policy Analysis

The legality of scraping public JavaScript for API keys has been tested in US courts. The Ninth Circuit Court of Appeals case **hiQ Labs v. LinkedIn (Case #17-16783)** established that scraping publicly accessible data does not violate the Computer Fraud and Abuse Act (CFAA) when access controls are not bypassed .

**Key Legal Principles for This Project** :

| Principle                    | Application to Qobuz Metadata Tagger                   |
| ---------------------------- | ------------------------------------------------------ |
| Public data only             | JS bundles are served publicly without authentication  |
| No access control bypass     | No password cracking or security circumvention         |
| Rate limiting required       | Prevents service disruption claims                     |
| Personal, non-commercial use | Reduces exposure to civil claims                       |
| No copyright republication   | Embeds metadata only, not audio content redistribution |

**Risk Mitigation**:
- Implement request delays (0.5–1 second between calls)
- Do not redistribute downloaded audio files
- Use for personal metadata enrichment only
- Respect `robots.txt` directives where present

### 2.3 Classical Music Metadata Handling

Classical music presents unique metadata challenges. Qobuz uses non-track items in API responses to represent movements and works .

**The Classical Gap Identified**:

```
Standard Track Structure (Pop):
Track 1: "Bohemian Rhapsody" (single work)

Classical Track Structure (Qobuz):
Track 1: "Piano Sonata No. 14 in C-sharp minor, Op. 27 No. 2 'Moonlight'"
  - Movement 1: "Adagio sostenuto" (track.work = entire sonata)
  - Movement 2: "Allegretto" (track.work = same sonata)
  - Movement 3: "Presto agitato" (track.work = same sonata)
```

**Implementation Strategy**:

```python
def extract_classical_metadata(track: dict) -> dict:
    """Handle Qobuz classical music metadata structure."""
    metadata = {}
    
    # Work grouping (e.g., "Symphony No. 5")
    if track.get("work"):
        metadata["work"] = track["work"]
        metadata["grouping"] = track["work"]  # Vorbis GROUPING
    
    # Movement title (e.g., "Allegro con brio")
    if track.get("version"):
        metadata["movement"] = track["version"]
        metadata["title"] = f"{track['work']}: {track['version']}"
    
    # Composer vs Performer distinction
    if track.get("composer"):
        metadata["composer"] = track["composer"]["name"]
    if track.get("performer"):
        metadata["performer"] = track["performer"]["name"]
    
    return metadata
```

```python
def embed_multi_disc_metadata(track_data: dict, audio_file: str) -> None:
    """
    Handle albums with multiple discs (media_number > 1).
    
    ID3v2.4 TPOS frame format: "disc_number/total_discs"
    Vorbis DISCNUMBER format: "disc_number/total_discs"

        tag = mutagen.id3.ID3(audio_file)
        tag.add(mutagen.id3.TPOS(text=disc_position))
    
    tag.save()
```

### 4.3 High-Res Artwork Embedding

**Critical Issue**: Embedding large images can corrupt audio stream padding if not handled correctly .

```python
def embed_artwork_safely(audio_file: str, image_url: str, max_size: int = 800) -> None:
    """
    Embed artwork without corrupting audio stream.
    
    Rules:
    1. Resize images >800px to prevent bloated files
    2. Always use JPEG format (best compatibility)
    3. Validate image before embedding
    """
    response = requests.get(image_url)
    img_data = response.content
    
    # Validate image header
    if not img_data.startswith(b'\xff\xd8'):  # JPEG SOI marker
        raise ValueError("Not a valid JPEG image")
    
    # Optional: Resize if too large
    if len(img_data) > 2_000_000:  # >2MB
        img = Image.open(io.BytesIO(img_data))
        img.thumbnail((max_size, max_size), Image.Resampling.LANCZOS)
        buffer = io.BytesIO()
        img.save(buffer, format='JPEG', quality=85, optimize=True)
        img_data = buffer.getvalue()
    
    # Embed using mutagen (FLAC)
    audio = mutagen.flac.FLAC(audio_file)
    audio.clear_pictures()  # Remove existing
    audio.add_picture(img_data, 'image/jpeg', 'Cover Art', mutagen.flac.PictureType.COVER_FRONT)
    audio.save()
```

## 5. Operational Safety & Fallbacks

### 5.1 Rate Limiting Implementation

Community consensus from successful projects establishes safe request limits to avoid temporary IP bans :

| Request Type   | Recommended Delay | Rationale                   |
| -------------- | ----------------- | --------------------------- |
| Album metadata | 1.0 seconds       | Heavy response payload      |
| Track metadata | 0.5 seconds       | Lighter, can be faster      |
| Search queries | 1.5 seconds       | Resource-intensive          |
| Download URLs  | 2.0 seconds       | Triggers streaming endpoint |

```python
import asyncio
import time
from functools import wraps

def rate_limit(delay_seconds: float):
    """Decorator for rate-limiting API calls."""
    def decorator(func):
        last_called = 0
        
        @wraps(func)
        def wrapper(*args, **kwargs):
            nonlocal last_called
            elapsed = time.time() - last_called
            if elapsed < delay_seconds:
                time.sleep(delay_seconds - elapsed)
            result = func(*args, **kwargs)
            last_called = time.time()
            return result
        return wrapper
    return decorator

class RateLimitedClient:
    def __init__(self, base_delay: float = 0.5):
        self.base_delay = base_delay
        self.request_times = []
    
    async def call_api(self, endpoint: str, params: dict):
        """Maintain sliding window of request times."""
        now = time.time()
        # Clean old entries (last 60 seconds)
        self.request_times = [t for t in self.request_times if now - t < 60]
        
        if len(self.request_times) >= 30:  # Max 30 requests per minute
            wait_time = 60 - (now - self.request_times[0])
            await asyncio.sleep(wait_time)
        
        self.request_times.append(now)
        return await self._make_request(endpoint, params)
```

### 5.2 Fallback Logic: MusicBrainz & MusicAPI.com

Qobuz metadata can be “patchy,” particularly for classical music where composition information may be missing or incorrectly formatted . Implement a cascading fallback strategy:

```python
class MetadataEnricher:
    """
    Cascade enrichment strategy:
    1. Primary: Qobuz API
    2. Secondary: MusicBrainz (via ISRC/UPC)
    3. Tertiary: MusicAPI.com (via title/artist)
    """
    
    def __init__(self):
        self.musicbrainz_client = MusicBrainzClient()
        self.musicapi_client = MusicAPIClient()
    
    async def enrich_track(self, qobuz_track: dict) -> dict:
        """Add missing metadata from alternative sources."""
        enriched = qobuz_track.copy()
        
        # Check for missing critical fields
        missing = []
        if not qobuz_track.get("isrc"):
            missing.append("isrc")
        if not qobuz_track.get("bpm"):
            missing.append("bpm")
        if not qobuz_track.get("work") and self._is_classical(qobuz_track):
            missing.append("work")
        
        if not missing:
            return enriched
        
        # Try MusicBrainz first (best for classical)
        if "isrc" in missing or "work" in missing:
            mb_result = await self.musicbrainz_client.lookup_by_isrc(
                qobuz_track.get("isrc")
            )
            if mb_result:
                enriched.update(mb_result)
        
        # Fallback to MusicAPI for BPM
        if "bpm" in missing:
            api_result = await self.musicapi_client.get_bpm(
                qobuz_track.get("title"),
                qobuz_track.get("artist", {}).get("name")
            )
            if api_result and api_result.get("bpm"):
                enriched["bpm"] = api_result["bpm"]
        
        return enriched
```

### 5.3 Error Recovery & Retry Strategy

```python
class ResilientAPIClient:
    """Implements exponential backoff for transient failures."""
    
    MAX_RETRIES = 3
    BASE_DELAY = 1.0
    
    async def request_with_retry(self, endpoint: str, params: dict):
        for attempt in range(self.MAX_RETRIES):
            try:
                response = await self.session.get(endpoint, params=params)
                
                if response.status_code == 400 and "InvalidAppSecretError" in response.text:
                    # Credentials expired - trigger refresh
                    await self.refresh_credentials()
                    continue
                
                if response.status_code == 429:  # Rate limited
                    wait_time = self.BASE_DELAY * (2 ** attempt)
                    await asyncio.sleep(wait_time)
                    continue
                
                response.raise_for_status()
                return response.json()
                
            except aiohttp.ClientError as e:
                if attempt == self.MAX_RETRIES - 1:
                    raise
                await asyncio.sleep(self.BASE_DELAY * (2 ** attempt))
        
        raise Exception(f"Failed after {self.MAX_RETRIES} retries")
```

## Appendix A: Complete Credential Extraction Example

```python
#!/usr/bin/env python3
"""
Complete credential extraction utility.
Based on vitiko98/qobuz-dl bundle.py and get_keys.py
Source: https://gist.github.com/vitiko98/bb89fd203d08e285d06abf40d96db592
"""

import re
import requests
from base64 import b64decode
from typing import Dict, Optional

class QobuzBundle:
    QOBUZ_URL = "https://play.qobuz.com"
    
    def __init__(self):
        self.session = requests.Session()
        self.session.headers.update({
            "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:133.0) Gecko/20100101 Firefox/133.0"
        })
    
    def get_app_id(self) -> str:
        bundle_js = self._fetch_bundle_js()
        match = re.search(r'appId\s*:\s*"(\d+)"', bundle_js)
        if not match:
            raise ValueError("Could not extract APP_ID")
        return match.group(1)
    
    def get_secrets(self) -> Dict[str, str]:
        bundle_js = self._fetch_bundle_js()
        pattern = r'initialSeed\("([^"]+)",window\.utimezone\.([^)]+)\)'
        matches = re.findall(pattern, bundle_js)
        
        secrets = {}
        for seed, tz in matches:
            try:
                decoded = b64decode(seed.encode()).decode('utf-8')
                secrets[tz] = decoded
            except Exception:
                continue
        
        return secrets
    
    def _fetch_bundle_js(self) -> str:
        # Get main page
        main_page = self.session.get(self.QOBUZ_URL).text
        
        # Find bundle.js URL
        bundle_pattern = r'src="([^"]+bundle\.js)"'
        bundle_match = re.search(bundle_pattern, main_page)
        if not bundle_match:
            raise ValueError("Could not locate bundle.js")
        
        bundle_url = f"https://play.qobuz.com{bundle_match.group(1)}"
        return self.session.get(bundle_url).text

# Usage
if __name__ == "__main__":
    bundle = QobuzBundle()
    app_id = bundle.get_app_id()
    secrets = bundle.get_secrets()
    
    print(f"APP_ID: {app_id}")
    print("\nAPP_SECRETS (try in order):")
    for tz, secret in secrets.items():
        print(f"  {tz}: {secret}")
```
