and either `ffmpeg` or `flac`) are not present.

## Remaining issues / next steps

* `ruff check .` reveals numerous style violations in legacy helper scripts (for
  example `dedupe_swap.py`, `temp_audio_dedupe_v3_1.py`, `verify_near_dupes.py`).
  They are unchanged here but still fail linting.
* The repair pipeline relies on `ffmpeg`; if the binary is missing the integration
  test and certain repair steps will skip/fail gracefully but the functionality is
  limited.
* Consider expanding automated coverage around the new diagnostics manager once
  a deterministic fixture strategy for external command outputs is available.

## Reproducing diagnostics inspection

To review the latest fingerprint dump without scanning:

```bash
python dd_flac_dedupe_db.py --fpcalc-dump-latest
python dd_flac_dedupe_db.py --check-fpcalc
```

These commands respect `--diagnostic-root` and will fall back to the runtime
directory if `/Volumes/dotad/.dedupe_diagnostics` is unavailable.
